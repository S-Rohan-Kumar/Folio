## 1. Product Overview

### 1.1 What is Pagebound?

Pagebound is a premium dark-mode mobile reading tracker that transforms solitary reading into a social, gamified, and AI-powered experience. It is comparable to Fable: Track & Discuss Books but goes significantly further with reading battles, mood-based discovery, personality profiling, and a Spotify Wrapped-style yearly review.

### 1.2 Target Users

- Avid readers who track their reading habits
- Book club members looking for a digital home
- Competitive readers who enjoy challenges
- Discovery-driven readers who want intelligent recommendations

### 1.3 Core Value Propositions

- **Track**: Know exactly where you are in every book, how fast you read, and how close you are to your annual goal.
- **Discover**: AI-powered recommendations tuned to your history, mood, and community trends.
- **Connect**: Public and private book clubs, live discussion threads, and social shelves.
- **Compete**: Challenge friends to reading battles — first to finish wins.
- **Reflect**: A yearly Reading Wrapped that celebrates your reading life.

### 1.4 Platform

- Flutter (Dart) — single codebase for iOS and Android
- Minimum iOS: 14.0
- Minimum Android: API 24 (Android 7.0)

## 3. Architecture

### 3.1 Pattern: Feature-First Clean Architecture

The app uses **Feature-First Clean Architecture** with Riverpod as the dependency injection and state layer. Each feature is fully self-contained.

```
Presentation Layer   →  Flutter Widgets + Riverpod Notifiers (UI state)
Domain Layer         →  Use Cases + Entities + Repository Interfaces
Data Layer           →  Repository Implementations + Remote/Local Data Sources
```

### 3.2 Data Flow (Unidirectional)

```
User Action
    │
    ▼
Widget calls Notifier method
    │
    ▼
Notifier calls Use Case
    │
    ▼
Use Case calls Repository Interface
    │
    ▼
Repository Implementation calls Data Source
    │
    ▼
Data Source returns raw data (JSON / Supabase response)
    │
    ▼
Repository maps to Domain Entity
    │
    ▼
Use Case returns Entity to Notifier
    │
    ▼
Notifier emits new AsyncValue state
    │
    ▼
Widget rebuilds via ref.watch()
```

### 3.3 Dependency Rule

- **Presentation** depends on **Domain** only (via interfaces)
- **Domain** has zero dependencies on outer layers
- **Data** depends on **Domain** interfaces and implements them
- **No circular dependencies** between features

### 3.4 Riverpod Provider Hierarchy

```dart
// Every feature follows this exact provider structure:

// 1. Data Source Provider (injectable, testable)
final bookRemoteDataSourceProvider = Provider<BookRemoteDataSource>((ref) {
  return BookRemoteDataSourceImpl(dio: ref.watch(dioProvider));
});

// 2. Repository Provider
final bookRepositoryProvider = Provider<BookRepository>((ref) {
  return BookRepositoryImpl(
    remote: ref.watch(bookRemoteDataSourceProvider),
    local: ref.watch(bookLocalDataSourceProvider),
  );
});

// 3. Use Case Provider
final searchBooksUseCaseProvider = Provider<SearchBooksUseCase>((ref) {
  return SearchBooksUseCase(ref.watch(bookRepositoryProvider));
});

// 4. Notifier (UI State)
final bookSearchNotifierProvider = AsyncNotifierProvider<BookSearchNotifier, List<Book>>(() {
  return BookSearchNotifier();
});
```

## 5. Database Schema

> All tables live in a single Supabase (Postgres) instance. Row-Level Security (RLS) must be enabled on every table. Policies are specified per table below.

### 5.1 `users` (extends Supabase auth.users)

```sql
CREATE TABLE public.users (
  id                  UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username            TEXT UNIQUE NOT NULL,
  display_name        TEXT,
  avatar_url          TEXT,
  bio                 TEXT,
  personality_type    TEXT,           -- e.g. "The Night Owl"
  personality_badge   TEXT,           -- emoji or icon code
  reading_goal_year   INT DEFAULT EXTRACT(YEAR FROM NOW()),
  reading_goal_target INT DEFAULT 12,
  xp                  INT DEFAULT 0,
  level               INT DEFAULT 1,
  streak_current      INT DEFAULT 0,
  streak_longest      INT DEFAULT 0,
  last_read_date      DATE,
  gemini_api_key      TEXT,           -- encrypted at rest, user-provided
  created_at          TIMESTAMPTZ DEFAULT NOW(),
  updated_at          TIMESTAMPTZ DEFAULT NOW()
);

-- RLS
ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can read all profiles" ON public.users FOR SELECT USING (true);
CREATE POLICY "Users can update own profile" ON public.users FOR UPDATE USING (auth.uid() = id);
```

### 5.15 Database Triggers

```sql
-- Auto-update books_finished in annual_goals when a book is marked finished
CREATE OR REPLACE FUNCTION update_annual_goal_counter()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = 'finished' AND OLD.status != 'finished' THEN
    INSERT INTO public.annual_goals (user_id, year, books_finished)
    VALUES (NEW.user_id, EXTRACT(YEAR FROM NOW()), 1)
    ON CONFLICT (user_id, year) DO UPDATE
    SET books_finished = annual_goals.books_finished + 1;
  ELSIF OLD.status = 'finished' AND NEW.status != 'finished' THEN
    UPDATE public.annual_goals
    SET books_finished = GREATEST(0, books_finished - 1)
    WHERE user_id = NEW.user_id AND year = EXTRACT(YEAR FROM NOW());
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_book_status_change
AFTER UPDATE ON public.user_books
FOR EACH ROW EXECUTE FUNCTION update_annual_goal_counter();

-- Auto-update streak on reading session creation
CREATE OR REPLACE FUNCTION update_reading_streak()
RETURNS TRIGGER AS $$
BEGIN
  UPDATE public.users
  SET 
    last_read_date = CURRENT_DATE,
    streak_current = CASE
      WHEN last_read_date = CURRENT_DATE - INTERVAL '1 day' THEN streak_current + 1
      WHEN last_read_date = CURRENT_DATE THEN streak_current
      ELSE 1
    END,
    streak_longest = GREATEST(streak_longest, CASE
      WHEN last_read_date = CURRENT_DATE - INTERVAL '1 day' THEN streak_current + 1
      ELSE 1
    END)
  WHERE id = NEW.user_id;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_reading_session_created
AFTER INSERT ON public.reading_sessions
FOR EACH ROW EXECUTE FUNCTION update_reading_streak();
```

```dart
double calculateReadingSpeed(List<ReadingSession> sessions) {
  if (sessions.isEmpty) return 0;
  final totalPages = sessions.fold(0, (sum, s) => sum + s.pagesRead);
  final totalHours = sessions.fold(0, (sum, s) => sum + s.durationSecs) / 3600;
  if (totalHours == 0) return 0;
  return totalPages / totalHours;
}

Duration estimateTimeRemaining(int pagesLeft, double pagesPerHour) {
  if (pagesPerHour == 0) return Duration.zero;
  final hoursLeft = pagesLeft / pagesPerHour;
  return Duration(minutes: (hoursLeft * 60).round());
}
```

### 7.6 AI Recommendation Engine (Gemini)

#### 7.6.1 API Setup

- Model: `gemini-2.0-flash`
- User provides their own Gemini Pro API key (stored in `users.gemini_api_key`, encrypted)
- API key entered in Settings → AI Recommendations → "Enter Gemini API Key"
- Key stored in Supabase with Postgres `pgcrypto` encryption:

```sql
-- Store encrypted
UPDATE public.users 
SET gemini_api_key = pgp_sym_encrypt('{key}', '{app_secret}')
WHERE id = '{user_id}';

-- Retrieve decrypted (in Edge Function only, never in client)
SELECT pgp_sym_decrypt(gemini_api_key::bytea, '{app_secret}') FROM public.users WHERE id = '{user_id}';
```

> **Architecture note:** The Gemini API call is made from a **Supabase Edge Function** — never directly from the Flutter client. The client sends a request to the Edge Function with the user's ID. The Edge Function fetches the decrypted key, builds the prompt, calls Gemini, and returns the recommendations. This prevents the API key from ever being exposed in network traffic from the device.

#### 7.6.2 Edge Function: `get-recommendations`

```typescript
// supabase/functions/get-recommendations/index.ts

import { serve } from 'https://deno.land/std/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js'

serve(async (req) => {
  const { userId } = await req.json()
  const supabase = createClient(Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!)

  // 1. Fetch user's reading history
  const { data: userBooks } = await supabase
    .from('user_books')
    .select('books(title, authors, categories), rating:reviews(rating)')
    .eq('user_id', userId)
    .eq('status', 'finished')
    .order('updated_at', { ascending: false })
    .limit(20)

  // 2. Fetch club's current read
  const { data: clubBooks } = await supabase
    .from('club_members')
    .select('clubs(current_book:books(title, authors, categories))')
    .eq('user_id', userId)

  // 3. Fetch decrypted API key
  const { data: user } = await supabase
    .rpc('get_decrypted_gemini_key', { user_id: userId })

  // 4. Build prompt
  const prompt = buildRecommendationPrompt(userBooks, clubBooks)

  // 5. Call Gemini
  const geminiResponse = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=${user.gemini_api_key}`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        contents: [{ parts: [{ text: prompt }] }],
        generationConfig: { temperature: 0.8, maxOutputTokens: 1024 }
      })
    }
  )

  const result = await geminiResponse.json()
  const text = result.candidates[0].content.parts[0].text

  // 6. Parse and return structured recommendations
  return new Response(JSON.stringify(parseRecommendations(text)), {
    headers: { 'Content-Type': 'application/json' }
  })
})
```

```sql
SELECT COUNT(*) FROM user_books 
WHERE user_id = $1 AND status = 'finished' AND EXTRACT(YEAR FROM finish_date) = $2;

-- Total pages
SELECT SUM(pages_read) FROM reading_sessions 
WHERE user_id = $1 AND EXTRACT(YEAR FROM session_date) = $2;

-- Top genre
SELECT unnest(b.categories) as genre, COUNT(*) as cnt
FROM user_books ub JOIN books b ON ub.book_id = b.id
WHERE ub.user_id = $1 AND ub.status = 'finished' AND EXTRACT(YEAR FROM finish_date) = $2
GROUP BY genre ORDER BY cnt DESC LIMIT 1;

SELECT SUM(duration_secs) / 3600.0 FROM reading_sessions
WHERE user_id = $1 AND EXTRACT(YEAR FROM session_date) = $2;
```

### 10.2 Notifier Pattern

```dart
// Every Notifier follows this structure:
@riverpod
class BookSearchNotifier extends _$BookSearchNotifier {
  @override
  FutureOr<List<Book>> build() => [];  // initial state

  Future<void> search(String query) async {
    if (query.isEmpty) {
      state = const AsyncValue.data([]);
      return;
    }
    state = const AsyncValue.loading();
    state = await AsyncValue.guard(() async {
      final useCase = ref.read(searchBooksUseCaseProvider);
      return useCase(SearchBooksParams(query: query));
    });
  }
}
```

### 10.3 AsyncValue Handling in UI

```dart
// Always handle all 3 states:
ref.watch(bookSearchNotifierProvider).when(

  data: (books) => books.isEmpty ? const EmptySearchState() : BookGrid(books: books),
);
```

```dart
factory Book.fromGoogleBooksJson(Map<String, dynamic> json) {
  final volumeInfo = json['volumeInfo'] as Map<String, dynamic>;
  final identifiers = (volumeInfo['industryIdentifiers'] as List? ?? []);
  
  return Book(
    id: json['id'],
    isbn10: identifiers.firstWhereOrNull((i) => i['type'] == 'ISBN_10')?['identifier'],
    isbn13: identifiers.firstWhereOrNull((i) => i['type'] == 'ISBN_13')?['identifier'],
    title: volumeInfo['title'] ?? 'Unknown Title',
    authors: List<String>.from(volumeInfo['authors'] ?? ['Unknown Author']),
    publisher: volumeInfo['publisher'],
    publishedDate: volumeInfo['publishedDate'],
    description: volumeInfo['description'],
    pageCount: volumeInfo['pageCount'],
    categories: List<String>.from(volumeInfo['categories'] ?? []),
    thumbnailUrl: (volumeInfo['imageLinks']?['thumbnail'] as String?)
        ?.replaceFirst('http://', 'https://'),  // Force HTTPS
    language: volumeInfo['language'] ?? 'en',
    averageRating: (volumeInfo['averageRating'] as num?)?.toDouble(),
  );
}
```

## 14. Security Requirements

### 14.1 API Key Security

- Gemini API key: stored encrypted in Supabase, decrypted only in Edge Functions
- Never log API keys or user tokens
- `gemini_api_key` column excluded from all SELECT * queries

### 14.2 Data Access

- Every Supabase table has RLS enabled (see schema, section 5)
- No client-side admin operations — all privileged DB access goes through Edge Functions with `service_role` key
- JWT validated on every Supabase request automatically

### 14.3 Input Validation

- All user-generated text fields: strip HTML, limit lengths as specified
- Club invite codes: regex `^[A-Z0-9]{8}$` validated server-side
- Page numbers: must be non-negative integers, bounded by book's page count
- Ratings: must be in range 0.5–5.0 in 0.5 increments

### 14.4 Content Moderation

- Reviews and thread posts: Supabase built-in text length limits
- Future: Edge Function calling Gemini to classify toxic content (flag for review, do not auto-delete)
